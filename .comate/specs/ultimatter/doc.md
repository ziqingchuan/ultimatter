# Ultimatter - 物质合成游戏 设计文档

## 项目概述

Ultimatter 是一个物质合成游戏，玩家通过组合已有物质来发现新物质。游戏提供基础合成模式和创意模式两种玩法，支持社区共创的合成树扩展。

## 技术栈

- **前端**: React + Vite + TypeScript
- **数据库**: Supabase（仅存储物质树信息，不存放涉密信息）
- **样式**: CSS Modules + 自定义炫酷暗色主题

## 核心概念

### 物质 (Matter)

物质属性：
- `name`: 物质名称（唯一，文本）
- `rarity`: 稀有度（1-10级，不同等级对应不同UI背景色）
- `synthesis_rules`: 合成规则（由哪些物质合成而来）
- `inventor`: 发明人（可选字段，创意模式中填写）

### 合成规则 (Synthesis Rule)

- 每条合成规则格式：`物质A + 物质B = 新物质`
- 即：一个新物质由恰好两个已有物质合成
- 基础物质（叶子节点）没有合成规则，是合成树的起点
- 合成规则存储为 `ingredient_a` + `ingredient_b` → `result`

## 游戏模式

### 基础合成模式

- 玩家从基础物质开始，通过拖拽/选择两个物质进行合成
- 系统查询数据库中匹配的合成规则，若有结果则获得新物质
- 获得的新物质加入玩家的"已发现物质"列表
- 若当前选中的两个物质没有合成结果，提示用户前往创意模式发明新规则
- 合成界面支持搜索/筛选已发现的物质
- 已发现物质面板以稀有度分组展示

### 创意模式

- 玩家从数据库现存全部物质中选择两个作为原料
- 实时检测合成规则冲突（输入时就检测，而非提交后）
  - 冲突1：该原料组合已有合成结果（即已存在 `A+B=C`）
  - 冲突2：新物质名称与已有物质重名
- 填写新物质的：名称（必填）、稀有度等级（必选）、发明人（选填）
- 提交后保存到 Supabase 数据库
- 提交成功后，新物质立即可用于后续合成和创意

## 数据库设计 (Supabase)

### 表结构

```sql
-- 物质表
CREATE TABLE matters (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  rarity SMALLINT NOT NULL CHECK (rarity >= 1 AND rarity <= 10),
  inventor TEXT,  -- 可选
  is_basic BOOLEAN DEFAULT FALSE,  -- 是否为基础物质（无合成规则）
  created_at TIMESTAMPTZ DEFAULT NOW()
);

-- 合成规则表
CREATE TABLE synthesis_rules (
  id UUID DEFAULT gen_random_uuid() PRIMARY KEY,
  ingredient_a UUID NOT NULL REFERENCES matters(id),
  ingredient_b UUID NOT NULL REFERENCES matters(id),
  result UUID NOT NULL REFERENCES matters(id),
  created_at TIMESTAMPTZ DEFAULT NOW(),
  UNIQUE(ingredient_a, ingredient_b)  -- 同一原料组合只能有一个结果
);
```

### RLS 策略

- 所有用户可读 matters 和 synthesis_rules
- 仅认证用户可插入新物质和合成规则（创意模式提交）
- 基础合成树数据由管理员预置

## UI 设计

### 整体风格

- 暗色科技风主题，深色背景 (#0a0a0f)
- 物质卡片带发光边框效果，边框颜色随稀有度变化
- 合成时有粒子动画效果
- 流畅的拖拽交互

### 页面结构

1. **首页/导航**: 游戏标题 + 模式选择入口
2. **基础合成模式页**:
   - 左侧：已发现物质面板（按稀有度分组，可搜索筛选）
   - 中间：合成台（两个物质槽 + 合成按钮 + 结果展示区）
   - 右侧：合成历史记录
3. **创意模式页**:
   - 上方：原料选择区（两个物质选择器，带搜索）
   - 中间：新物质信息填写区（名称、稀有度、发明人）
   - 实时冲突检测提示区
   - 下方：提交按钮
4. **物质图鉴页**: 展示所有已发现/全部物质，按稀有度排列

### 交互流程

**基础合成**:
1. 从左侧面板拖拽或点击选择物质放入合成槽
2. 两个槽都有物质后，合成按钮亮起
3. 点击合成，查询数据库匹配规则
4. 成功：结果物质以动画形式展示，加入已发现列表
5. 失败：提示"未发现合成路径，前往创意模式发明？"

**创意模式**:
1. 选择原料A（从搜索列表中选）
2. 选择原料B（从搜索列表中选）
3. 每次选择变化时，实时检测是否已有该组合的合成结果
4. 填写新物质名称（输入时实时检测是否重名）
5. 选择稀有度等级
6. 可选填写发明人
7. 全部通过验证后提交按钮可用
8. 提交成功后显示成功反馈

## 数据流

1. **初始化**: 从 Supabase 加载全部物质和合成规则到前端状态
2. **基础合成**: 用户选择两个物质 → 本地查找合成规则 → 展示结果
3. **创意模式**: 用户选择原料+填写信息 → 实时校验 → 提交到 Supabase → 更新本地状态
4. **冲突检测**: 选择原料时即查询 synthesis_rules 表是否已有该组合

## 边界条件与异常处理

1. **合成规则冲突**: 原料组合已存在结果 → 在原料选择阶段立即提示
2. **物质重名**: 名称输入时实时校验 → 即时提示
3. **空输入校验**: 名称、稀有度为必填，提交前校验
4. **网络异常**: Supabase 请求失败时提示用户重试
5. **自身合成**: 不允许 A+A 的合成（需两个不同物质）
6. **循环依赖**: 合成规则不会产生循环（因为合成只能由已知物质产生新物质，逻辑上不会循环）
